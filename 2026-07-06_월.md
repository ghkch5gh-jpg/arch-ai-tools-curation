<section class="news-flow-sec reveal"><h2 class="news-flow__t">이번 레시피 흐름</h2><div class="news-flow"><p>이번 레시피는 '왜 이 매스인가'를 숫자로 설명할 수 있는 컨셉 스터디 세트를 만듭니다. 대지 polyline·용적·높이를 파라미터로 넣어 그래스호퍼(GH)로 매스 대안을 자동 생성하고, Ladybug Tools로 일조·조망을 평가한 뒤, Wallacei 진화 솔버(여러 목표를 동시에 저울질하는 자동 탐색기)로 후보를 추리고, 상위안을 비교 다이어그램으로 정리합니다. 도구 체인은 Food4Rhino(플러그인 수급) → Wallacei(최적화) → Ladybug Tools(환경분석)이고, 셋 다 무료지만 GH·진화알고리즘 이해가 필요한 심화 과정입니다. 현상 초기 매싱 스터디를 하는 실무자·대학원생용이며, 플러그인 설치·정의 셋업에 반나절, 이후 최적화 연산에 수십 분~수 시간이 걸립니다.</p></div></section>

<section class="news-sec">
<h2 class="news-sec__t">단계</h2>
<article class="ni reveal">
  <header class="ni__h">
    <span class="ni__n">01</span>
    <span class="ni__t">플러그인 수급 + Rhino8 호환 확인</span>
    <span class="ni__tr"><span class="ni__score">약 30~40분</span></span>
  </header>
  <p class="ni__body">Food4Rhino에서 Wallacei와 Ladybug Tools를 각각 내려받아 GH에 설치합니다. .gha 파일 차단이 흔하니 파일 속성에서 '차단 해제(Unblock)'를 먼저 하고, Rhino를 재시작해 컴포넌트 탭이 뜨는지 확인합니다. Rhino8 사용자는 각 플러그인의 최신 빌드가 Rhino8 지원인지(특히 Ladybug의 파이썬 백엔드) 릴리스 노트를 꼭 확인하세요.</p>
  <div class="ni__meta"><div class="ni__row ni__row--do"><dt>팁</dt><dd class="ni__do-text">설치 후 GH에서 컴포넌트가 회색/누락이면 대개 Unblock 누락이거나 버전 불일치입니다 — 재설치보다 먼저 이 둘부터 의심하세요.</dd></div></div>
</article>
<article class="ni reveal">
  <header class="ni__h">
    <span class="ni__n">02</span>
    <span class="ni__t">대지·용적·높이 파라미터 정의</span>
    <span class="ni__tr"><span class="ni__score">약 60~90분</span></span>
  </header>
  <p class="ni__body">대지 polyline을 Rhino에서 GH로 참조하고, 이격거리·건폐율·용적률·최고높이·층고를 각각 슬라이더(Number Slider)로 노출합니다. 대지 경계를 오프셋해 건축가능 영역을 만들고, 그 안에서 매스 풋프린트가 결정되도록 로직을 짭니다. 이 슬라이더들이 뒤 단계에서 Wallacei가 흔들 '유전자(변수)'가 되므로, 값 범위(min/max)를 현실적인 법규·실무 범위로 잡아두세요.</p>
  <div class="ni__meta"><div class="ni__row ni__row--do"><dt>팁</dt><dd class="ni__do-text">변수는 처음부터 많이 열지 마세요. 3~5개로 시작해야 최적화 탐색공간이 감당 가능하고 결과 해석도 됩니다.</dd></div></div>
</article>
<article class="ni reveal">
  <header class="ni__h">
    <span class="ni__n">03</span>
    <span class="ni__t">GH로 매스 대안 생성 로직 구성</span>
    <span class="ni__tr"><span class="ni__score">약 60~90분</span></span>
  </header>
  <p class="ni__body">풋프린트를 Extrude·Cap으로 매스화하고, 셋백/보이드/코어 위치 같은 조작을 파라미터로 연결해 하나의 정의가 슬라이더 값에 따라 서로 다른 매스를 뱉게 만듭니다. 층수·연면적을 실시간으로 계산해 패널로 띄워, 슬라이더를 손으로 움직이며 로직이 깨지지 않는지(교차·자기관통 등) 먼저 눈으로 검증합니다.</p>
  <div class="ni__meta"><div class="ni__row ni__row--do"><dt>팁</dt><dd class="ni__do-text">최적화를 걸기 전에 극단값(모든 슬라이더 min, 모든 max)에서 지오메트리가 유효한지 확인하세요. 여기서 터지면 최적화 도중 통째로 멈춥니다.</dd></div></div>
</article>
<article class="ni reveal">
  <header class="ni__h">
    <span class="ni__n">04</span>
    <span class="ni__t">Ladybug로 일조·조망 평가 지표 연결</span>
    <span class="ni__tr"><span class="ni__score">약 60분</span></span>
  </header>
  <p class="ni__body">국내 대지 위치의 EPW 기상데이터를 확보해 Ladybug에 물리고, 각 매스에 대해 일조시간(sunlight hours)·주변 조망 개방도 같은 지표를 계산하는 컴포넌트를 붙입니다. 계산 결과가 단일 숫자(예: 평균 일조시간)로 요약돼 나오게 만들어, 다음 단계의 최적화 목표로 바로 쓸 수 있게 합니다.</p>
  <div class="ni__meta"><div class="ni__row ni__row--do"><dt>팁</dt><dd class="ni__do-text">국내 EPW는 별도로 구해야 합니다. 정밀 분석 전에는 그리드 해상도를 거칠게 잡아 연산시간을 줄이고, 로직이 맞는지부터 확인하세요.</dd></div></div>
</article>
<article class="ni reveal">
  <header class="ni__h">
    <span class="ni__n">05</span>
    <span class="ni__t">Wallacei 다목적 최적화 실행</span>
    <span class="ni__tr"><span class="ni__score">약 30분~수 시간 (설정·연산 규모에 따라)</span></span>
  </header>
  <p class="ni__body">Ladybug 지표(예: 일조 최대화)와 면적·형상 지표(예: 목표 연면적과의 오차 최소화)를 Wallacei의 목표(objective)로 연결하고, 앞서 만든 슬라이더들을 유전자로 지정합니다. 세대 수(generation)와 개체 수(population)를 작게(예: 20×20 수준) 시작해 시험 run을 돌려 수렴 경향을 본 뒤 본 연산을 겁니다.</p>
  <div class="ni__meta"><div class="ni__row ni__row--do"><dt>팁</dt><dd class="ni__do-text">지표는 컨셉 근거일 뿐 성능 인증이 아닙니다. Wallacei가 뽑은 '최적'은 당신이 준 목표함수 안에서의 최적이라, 목표 설정이 곧 결과를 좌우합니다.</dd></div></div>
</article>
<article class="ni reveal">
  <header class="ni__h">
    <span class="ni__n">06</span>
    <span class="ni__t">상위안 추림 + 비교 다이어그램 정리</span>
    <span class="ni__tr"><span class="ni__score">약 60~90분</span></span>
  </header>
  <p class="ni__body">Wallacei의 파레토 프론트(절충 최적안 집합)와 목표별 분포 그래프에서 상위 3~4안을 골라 GH에서 지오메트리를 복원(bake)하고, 각 안의 일조시간·연면적·형상 지표를 표/다이어그램으로 나란히 정리합니다. 각 안이 '무엇을 얻고 무엇을 포기했는지'를 한 장에 보이게 배치해 채택 근거 슬라이드로 만듭니다.</p>
  <div class="ni__meta"><div class="ni__row ni__row--do"><dt>팁</dt><dd class="ni__do-text">패널에는 '최적'이라는 단정 대신 '이 목표 기준 상위안'으로 쓰세요. 심사에서 지표 정의를 되물으면 방어 가능해야 합니다.</dd></div></div>
</article>
</section>
