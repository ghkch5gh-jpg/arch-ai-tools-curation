<section class="news-flow-sec reveal"><h2 class="news-flow__t">이번 레시피 흐름</h2><div class="news-flow"><p>라이노에서 확정된 입면 뷰 하나를 깊이맵·외곽선으로 변환해 ControlNet으로 형상을 잠근 뒤, 프롬프트만 바꿔 재질·시간대·분위기가 다른 입면 변주안 여러 컷을 만드는 레시피입니다. 체인은 Stable Diffusion+ControlNet → ComfyUI(노드 파이프라인) → FLUX(대안 모델). SD·DD 단계에서 클라이언트에게 '같은 형상, 다른 표정'을 빠르게 보여줘야 하는 실무자용입니다. 로컬 GPU만 있으면 생성 비용은 0이지만, 첫 셋업(ComfyUI 설치·VRAM 12GB)과 노드 학습이라는 진입장벽이 시간 비용으로 들어옵니다 — 그래서 이번 회차는 '실행 보고'가 아니라 따라하기 가이드입니다.</p></div></section>
<section class="news-sec">
<h2 class="news-sec__t">재료 — 쓰는 도구</h2>
<article class="ni reveal">
  <header class="ni__h">
    <span class="ni__n">01</span>
    <a class="ni__t" href="https://github.com/lllyasviel/ControlNet" target="_blank" rel="noopener">Stable Diffusion + ControlNet <span class="ni__arrow">↗</span></a>
    <span class="ni__tr"><span class="ni__src">재료</span></span>
  </header>
  <p class="ni__body">라이노 뷰의 깊이맵·외곽선(canny)을 '조건'으로 걸어 형상을 정확히 고정하는 핵심. ControlNet은 그림의 윤곽선만 따라 그리게 잡아주는 가이드레일이라고 보면 됩니다 — 형상은 그대로, 채색만 AI가 다시.</p>
  <ul class="ni__pts"><li>오픈소스(무료, 로컬). Apache-2.0 라이선스</li><li>로컬 GPU (PyTorch, Windows/Mac/Linux), HuggingFace 가중치</li></ul>
</article>
<article class="ni reveal">
  <header class="ni__h">
    <span class="ni__n">02</span>
    <a class="ni__t" href="https://www.comfy.org/" target="_blank" rel="noopener">ComfyUI <span class="ni__arrow">↗</span></a>
    <span class="ni__tr"><span class="ni__src">재료</span></span>
  </header>
  <p class="ni__body">ControlNet·업스케일·인페인트를 노드로 엮어 '라이노 뷰 → 형상고정 렌더 → 패널 해상도'를 재현 가능한 파이프라인으로 고정. 한 번 짜두면 시리즈 컷을 일관되게 양산합니다.</p>
  <ul class="ni__pts"><li>오픈소스(무료, 로컬·데스크톱) + 클라우드/엔터프라이즈 유료 옵션</li><li>로컬 데스크톱 + 클라우드 + API (노드 기반 SD 파이프라인)</li></ul>
</article>
<article class="ni reveal">
  <header class="ni__h">
    <span class="ni__n">03</span>
    <a class="ni__t" href="https://blackforestlabs.ai/" target="_blank" rel="noopener">FLUX (Black Forest Labs) <span class="ni__arrow">↗</span></a>
    <span class="ni__tr"><span class="ni__src">재료</span></span>
  </header>
  <p class="ni__body">SD 대신 쓸 수 있는 오픈 웨이트 이미지 모델. 같은 ComfyUI·ControlNet 그래프에 끼워 재질 표현이 더 또렷한 변주가 필요할 때 교체합니다. 제출물에 상업 사용 시 라이선스 조건 확인 필수.</p>
  <ul class="ni__pts"><li>오픈 웨이트(무료, 비상업 라이선스) + 상업용/자체호스팅 라이선스 별도 + 클라우드 API 유료. 플레이그라운드 무료</li><li>로컬 GPU (오픈 웨이트, HuggingFace/GitHub) + 클라우드 API. 현재 FLUX.2 세대</li></ul>
</article>

<section class="news-sec">
<h2 class="news-sec__t">단계</h2>
<article class="ni reveal">
  <header class="ni__h">
    <span class="ni__n">01</span>
    <span class="ni__t">라이노 입면 뷰·카메라 고정</span>
    <span class="ni__tr"><span class="ni__score">약 5분</span></span>
  </header>
  <p class="ni__body">변주할 입면 하나를 정해 라이노에서 평행투영(혹은 약한 원근)으로 카메라를 잡고 Named View로 저장합니다. 이후 모든 컷이 같은 각도로 나와야 비교가 되니, 이 뷰는 레시피 끝까지 절대 건드리지 않습니다. 해상도는 가로 1024~1536px 정도로 뷰포트 캡처를 미리 맞춰둡니다.</p>
  <div class="ni__meta"><div class="ni__row ni__row--do"><dt>팁</dt><dd class="ni__do-text">배경·주변 맥락은 일단 비우고 매스만 깔끔하게 — 형상 조건이 지저분하면 AI가 엉뚱한 디테일을 채웁니다.</dd></div></div>
</article>
<article class="ni reveal">
  <header class="ni__h">
    <span class="ni__n">02</span>
    <span class="ni__t">깊이맵·외곽선 맵 추출</span>
    <span class="ni__tr"><span class="ni__score">약 10분</span></span>
  </header>
  <p class="ni__body">같은 뷰에서 두 종류의 조건 이미지를 뽑습니다. (1) 깊이맵 — 라이노의 ClippingPlane/ZBuffer 계열 디스플레이나 렌더의 Z-depth 패스로 앞뒤 거리감을, (2) 외곽선 — Make2D 또는 Technical 디스플레이 모드로 선만. 캐니(canny)는 이 외곽선을 AI가 읽는 형식이라고 보면 됩니다.</p>
  <div class="ni__meta"><div class="ni__row ni__row--do"><dt>팁</dt><dd class="ni__do-text">깊이맵은 '입면 깊이감·요철', 캐니는 '선·창호 분할'에 강합니다. 둘 다 만들어두고 단계 4에서 무엇을 더 강하게 걸지 고릅니다.</dd></div></div>
</article>
<article class="ni reveal">
  <header class="ni__h">
    <span class="ni__n">03</span>
    <span class="ni__t">ComfyUI에 형상 고정 그래프 구성</span>
    <span class="ni__tr"><span class="ni__score">약 20~40분 (첫 셋업이면 더)</span></span>
  </header>
  <p class="ni__body">ComfyUI를 실행하고 App Mode의 ControlNet 템플릿에서 시작합니다. Load Image(조건 맵) → ControlNet Apply(depth 또는 canny) → KSampler → VAE Decode 흐름으로 연결하고, ControlNet strength를 0.7~1.0 사이로 잡아 형상을 강하게 잠급니다. 처음이면 커뮤니티 템플릿을 불러와 노드만 바꿔 끼우는 게 빠릅니다.</p>
  <div class="ni__meta"><div class="ni__row ni__row--do"><dt>팁</dt><dd class="ni__do-text">한 번 완성한 그래프는 .json으로 저장 — 다음 프로젝트에선 조건 이미지만 교체하면 그대로 재사용됩니다.</dd></div></div>
</article>
<article class="ni reveal">
  <header class="ni__h">
    <span class="ni__n">04</span>
    <span class="ni__t">프롬프트로 재질·시간대·분위기 변주</span>
    <span class="ni__tr"><span class="ni__score">약 15분 (생성 시간은 GPU에 따라)</span></span>
  </header>
  <p class="ni__body">형상은 ControlNet이 잡고 있으니, 프롬프트만 바꿔 변주를 만듭니다. 예: '주광·노출 콘크리트', '석양·테라코타 루버', '흐린 오전·유리 커튼월'처럼 시간대+재질을 한 줄씩. seed를 고정하면 같은 구도에서 재질만 비교, seed를 바꾸면 같은 조건의 다른 해석을 얻습니다. 한 조건당 3~4컷씩 배치로 돌립니다.</p>
  <div class="ni__meta"><div class="ni__row ni__row--do"><dt>팁</dt><dd class="ni__do-text">한 번에 한 변수만 바꾸기 — 재질과 시간대를 동시에 바꾸면 어느 프롬프트가 효과였는지 추적이 안 됩니다.</dd></div></div>
</article>
<article class="ni reveal">
  <header class="ni__h">
    <span class="ni__n">05</span>
    <span class="ni__t">FLUX로 교체 테스트(선택)</span>
    <span class="ni__tr"><span class="ni__score">약 15분</span></span>
  </header>
  <p class="ni__body">SD 결과의 재질 디테일이 아쉽다면 같은 그래프에서 모델만 FLUX 오픈 웨이트로 바꿔 동일 조건으로 다시 돌려 비교합니다. FLUX는 재질·질감 표현이 또렷한 편이라 클로즈업 패널 컷에 유리할 수 있습니다. 단, 제출물에 쓸 거면 비상업/상업 라이선스 조건을 먼저 확인합니다.</p>
  <div class="ni__meta"><div class="ni__row ni__row--do"><dt>팁</dt><dd class="ni__do-text">무료로 감을 먼저 보려면 BFL 플레이그라운드(playground.bfl.ai)에서 프롬프트만 빠르게 검증한 뒤 로컬로 옮기는 방법도 있습니다.</dd></div></div>
</article>
<article class="ni reveal">
  <header class="ni__h">
    <span class="ni__n">06</span>
    <span class="ni__t">업스케일·패널 정리</span>
    <span class="ni__tr"><span class="ni__score">약 15분</span></span>
  </header>
  <p class="ni__body">고른 컷을 ComfyUI의 업스케일 노드로 패널 해상도까지 키우고, 어긋난 부분은 인페인트 노드로 부분 수정합니다. 최종 N안을 같은 뷰·같은 크기로 나란히 배치하면 '형상 동일, 표정만 다른' 비교 시트가 됩니다.</p>
  <div class="ni__meta"><div class="ni__row ni__row--do"><dt>팁</dt><dd class="ni__do-text">업스케일은 마지막에 한 번만 — 중간 단계에서 키워두면 배치 생성이 느려지고 VRAM이 터집니다.</dd></div></div>
</article>
</section>
